Smash Stack Protector

When you build a package dsrc will automatically enable the Smash Stack Protector for packages that support it. The SSP extension consists from two flags, -fstack-protector and -fstack-protector-all which are passed to the compiler with the CFLAGS and CXXFLAGS flags.

A short explanation of these flags (from the gcc man page):
Generate code to protect an application from a stack smashing attack. The features are (1) the insertion of random value next to the frame pointer to detect the integrity of the stack, (2) the reordering of local variables to place buffers after pointers to avoid the corruption of pointers that could be used to further corrupt arbitrary memory locations, (3) the copying of pointers in function arguments to an area preceding local variable buffers to prevent the corruption of pointers that could be used to further corrupt arbitrary memory locations, and the (4) omission of instrumentation code from some functions to decrease the performance overhead. If the integrity would be broken, the program is aborted. If no-stack-protector is specified, instrumentation codes are generated at every functions.

So using these flags will improve the security of the system at the cost of unnotable performance lost. If you plan to use dsrc on a system that doesnt have the SSP expension installed, then you should download the patch protector-3.3.2-3.tar.gz and install it as explained on the Smash Stack Protector site.
