Building packages

Until now i have explained how to maintain package in precompiled form, there is another way too, you can build your own packages and install them. Is this useful? Yes it's very useful. Lets say you want to recompile the entire system and optimize it for you CPU, or you want to install packages that are not available in precompiled form, or you want to add options to your packages. Until now there are about 900 packages that can be built from sources, but the list is growing with every day.

How is this done? So each package that can be built from sources has a build file which contains information about the package that is about to be built. You will learn how to write your own package builts here.
