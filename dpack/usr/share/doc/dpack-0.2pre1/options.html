Options

Options that can be used when building packages:
[root@localhots:~]# dsrc -h
usage: dsrc [options] <argument1> <argument2> ...
options:
  -b, --builddeps        - build package dependencies from source
  -c, --noclean          - do not remove work files
  -C, --continue         - use work files (continue with them) (dangerous)
  -d, --nodeps           - skip package dependencies checks
  -D, --download         - only download sources, do not compile them
  -e, --sysinfo          - print information about dsrc
  -f, --force            - overwrite existing package
  -g, --genmd5           - genearate MD5 messages
  -i, --noinstall        - do not install package
  -I, --info <package>   - print package information (from build file)
  -k, --keep             - keep created package
  -n, --nostrip          - do not strip binaries and libraries
  -o, --optimize         - optimize code for running system
  -p, --nossp            - do not use Smash Stack Protector (Propolice)
  -P, --pretend          - only show actions, do not run them (not working)
  -r, --rmdeps           - remove installed dependencies
  -R, --rebuild          - rebuild package (even if package is installed)
  -s, --syncdeps         - install package dependencies using dupdate
  -S, --search           - search <package> or <expression> in database
  -u, --upgrade          - upgrade installed packages (used for groups)
  -w, --write <dir>      - save package to <dir>
  -v, --verbose          - be more verbose
  -V, --version          - print name, version and license
  -H, --howto            - a short howto with some examples
  -h, --help             - print this message
arguments:
   <package>  - build/rebuild the package
   <group>    - build/rebuild entire group
   <sync>     - sync package builds database
   <world>    - build/rebuild the world

For a detailed help please see dpack (8).
[root@localhost:~]#

And the detailed version here:

-b, --builddeps

    normally dsrc will check if dependencies are satisfied before building a package, and exit with and error message (with the missing dependencies). Using this option will make dsrc try to build missing dependencies before building the package.
-c, --noclean
    when you build a package the source files will be unpackaged in the SRC directory (usually $PACKAGESDIR/$THREE/group/name/SRC, where PACKAGESDIR and THREE are defined in /etc/dpack/build) and the package contents are saved in the PKG directory (usually $PACKAGESDIR/$THREE/group/name/PKG), but these two directories are removed after a successfull build. Using this option will make dsrc keep these directories.
-C, --continue
    if a package build faills or you stop it with CTRL+C, then you can use this option to continue the build, with the sources available in $PACKAGESDIR/$THREE/group/name/SRC (since the sources are only kept when you use -c option, the -C option is only usefull when you use it together with the -c option).
-d, --nodeps
    before building a package dsrc will check if all dependencies are satisfied, using this option will make dsrc skip this step. This option should only be used if you know what you doing and since allmost all packages have dependencies, they will not build correctly or not build at all if you use this option.
-D, --download
    this option will make dsrc download the source files and save them in $CACHEDIR which is defined in /etc/dpack/common (usually /var/cache/dsrc) but it will not build the package. Usefull if you want to download the sources and compile whenever you want.
-f, --force
    if you want to build a package that has already been built (with the -k option) and saved in $PACKAGESDIR/$THREE/group/name/package/name-version-arch-release.tbz2 the dsrc will exit with a warning message (eg: [WARNING]: package name-version-arch-release.tbz2 already built.). If you want to rebuild the package and overwite the saved one use this option.
-g, --genmd5
    every package has its own integrity file (a file containing md5sum messages for earch source file), which is generated by the packager, so if you want build your own packages you sould use this option. It can also be used if you want to build a package that didn't pass the integrity check.
-i, --noinstall
    normally dsrc will build the package, and if this operation is successsull it will install it too, using this option will make dsrc build the package but dont install it
-I, --info
    this option will print information about a package (the information is gathered from the build file) and will contain the following variables: latest available package, installed package, license, homepage, description, download location, size, time needed to build the package (depending on the bogomips of your CPU) and package dependencies.
-k, --keep
    after you build a package dsrc will install it and delete the built packagem, if you use this option the package will be saved in $PACKAGESDIR/$THREE/group/name/package/name-version-arch-release.tbz2
-n, --nostrip
    the files (only libraries and binaries) of a package are striped before building it, if you use this option dsrc will not strip the files. This is usefull if you want to debug the created code (in most cases strip will remove the debug functions from the binary).
-o, --optimize
    this option will overide the compiler optimization flags (from /etc/dpack/build), and add optimized flags for your system. A more detailed explanation for this option can be found here: Optimizing build
-p, --nossp
    this option will deactivate the gcc Smash Stack Protector Extension (formelly know as Propolice). There are only a few packages that dont build with this extension (glibc, xorg and kernel-*) and dsrc will automatically deactive SSP for these packages, so this option should only be used if you really dont want SSP support. This option is only provided for the user's freedom of choice and here you can find information about SSP and why you should use this option everytime it is possible.
-P, --pretend
    this option will show actions that the command will do but dont execute them (this is usefull for testing).
-r, --rmdeps
    when dsrc installs a package and you provided the -b or -s option, then if you provide this option it will remove the installed dependecies (it doesnt matter if they were installed with dsrc or dupdate). This option should be used only if you really know what you are doing, the package will 100% not work if the dependencies are removed.
-R, --rebuild
    if you want to build a package that is already installed (same version and release) then dsrc will exit with an error message (eg: package NAME already installed (NAME-VERSION)). Use this option if you really want rebuild/reinstall the package. Usefull if you want to optimize your system.
-s, --syncdeps
    this option will tell dsrc to install missing dependencies with dupdate. Usefull if you want to install a package that is not available in a precompiled from. More informations about dupdate can be found here.
-S, --search
    use this option if you want for to search for a package in the builds database, but you dont know the exactly name of the package (it will search for packages containing *argument*)
-u, --update
    if you want to build/install a package and you want the latest version you should use this option. It will get the latest build file from $RSYNCMIRROR (defined in /etc/dpack/build).
-w, --write
    nomally dsrc will save the package (if used together with the -k option) in $PACKAGESDIR/$TREE/group/name/package, if you want to save the package to another directory the use this option.
-v, --verbose
    this tells dsrc to be more verbose. More information about this option can be found here.
-H, --howto
    you can allways read this entire page in a text version if you provide this option.
The arguments are:
<package>
    this should be pretty self explanatory (just in case its only the name of the package)<
<group>
    this is the name of a group. At the moment of the writing following groups are available:
    base, chat, devel, doc-tools, editor, games, gnome, gnome-extra, graphics, kde, kde-extra, kde-i8n, kernels, libs, mail, multimedia, network, office, perl-modules, printing, security, system, web, x11, xfce and xfce-extra. If you provide a group as an argument dsrc will build the entire group and use the options described on this page.
<sync>
    if there are changes in the builds database (new builds are added or security updates) the you can use this option argument to sync the builds database. If will use rsync to fetch the latest build files.
<world>
    now this option will really let your rebuild the world as you want.Let say you want to uptimize your system, or you want builds to use specific configurations (provided in /etc/dpack/build as USE variables), the you should really use this option. It will use the options described here for each package it builds/rebuilds.
Some important locations (defaults):
/usr/src/packages/release/                   - the package database builds
/usr/src/packages/realase/name/            - the directory containing the build and verify files
/usr/src/packages/release/name/files/    - the directory where patches, initscripts and configuration files
/usr/src/packages/release/name/package/ - the directory where the package is saved
/usr/src/packages/release/name/SRC/       - the unpacked (compiled) sources
/usr/src/packages/release/name/PKG/       - the unpacked package contents
/var/cache/dsrc/                                 - here are the sources
