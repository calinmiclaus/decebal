Installing packages

To install packages you use pkg_install.

[root@localhost:~]# pkg_install -h
usage: pkg_install [options] <package>
options:
  -d, --nodeps       - skip all dependencies checks (dangerous)
  -f, --force        - force installation, overwrite existing files
  -r, --root <path>  - use an alternative installation path
  -v, --verbose      - be more verbose
  -V, --version      - print name, version and license
  -h, --help         - print this message and exit
[root@localhost:~]#

The detailed explanation of the options is here:

-d,--nodeps

    normally pkg_install will check if all the packages needed by the package that is about
    be installed, are present, if not it will print an error message and exit. Using this option will make pkg_install skip this step.
-f, --force
    force installation, overwrite existing files. If the package that is about to be installed files that are already installed, pkg_install will print an error message (with the existing files) and exit. Using this option will make pkg_install skip this step.
-r, --root <path>
    use an alternate <path> for installation. This option should be used if you want to install packages on another system (a amounted partition) and not for installing packages in another prefix (like /usr or /usr/local).
-v, --verbose
    be more verbose, use this option if you want pkg_install to print more information (without this option pkg_install will only print a one line message)
Examples:

Installing a package:
[root@localhost:~]# pkg_install ed-0.2-i686-1.tbz2
[+] installing ed...[OK]
[root@localhost:~]#


Installing a package (verbose mode):
[root@localhost:~]# pkg_install -v ed-0.2-i686-1.tbz2
[+] installing ed...

Installing a package (and printing dependencies errors):
[root@localhost:~]# pkg_install irssi-0.0.10_rc5-i686-1.tbz2
[+] installing irssi...[FAILED]
[ERROR]: missing dependencies:
     glib2
[root@localhost:~]#

Installing same package (with -d options):
[root@localhost:~]# pkg_install -d irssi-0.0.10_rc5-i686-1.tbz2
[+] installing irssi...[OK]
[root@localhost:~]#

The package has beed installed, but this doesnt mean that it also works, if i run irssi now, it will exit with an error (it cant load glib2 shared library).
This option should only be used for testing, or if you plan to install the missing dependencies after you installed the package.
