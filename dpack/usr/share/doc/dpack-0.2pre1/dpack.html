Using dpack

This program (dpack) is actually a wrapper for pkg_add, pkg_del pkg_info, dupdate and dsrc.

A quick help for dpack:
[root@localhost:~]# dpack -h
usage: dpack <action> [options] <argument>
actions:
  dpack {-B, --build}   [options] <argument>   - build/install packages from sources (dsrc)
  dpack {-I, --install} [options] <package>    - install a package (pkg_install)
  dpack {-R, --remove}  [options] <package>    - remove a package (pkg_remove)
  dpack {-S, --sync}    [options] <argument>   - update a package (dupdate)
  dpack {-U, --upgrade} [options] <package>    - upgrade a package (pkg_upgrade)
  dpack {-Q, --query}   [options] <argument>   - query the package database (pk_query)
  dpack {-V, --version}                        - print name, version and license
  dpack {-h, --help}                           - print this message and exit
options: are the options corresponding to the respective tool.

NOTE: you can only use one option at a time.

For a detailed help see dpack (8).
[root@localhost:~]#


Some examples:

Installing a package with dpack:
[root@localhost:~]# dpack -I ed-0.2-i686-1.tbz2
[+] installing ed...[OK]
[root@localhost:~]#

Building and installing a package with dpack:
[root@localhost:~]# dpack -B ed
[+] building ed...
    [-] checking dependencies...[OK]
    [-] fetching source files...[OK]
    [-] checking source files integrity...[PASSED]
    [-] extracting source files...[DONE]
    [-] running build...[DONE]
    [-] creating package ed-0.2-i686-1.tbz2...[OK]
[+] installing ed...[OK]
[root@localhost:~]#

These are a few examples (you can read about dsrc in the next chapter), but i hope you got the idea.
