Removing packages

To remove packages you use pkg_remove.

[root@localhost:~]# pkg_remove -h
usage: pkg_add [options] <package>
options:
  -b, --nobackup     - remove configuration files as well
  -d, --nodeps       - skip all dependencies checks (dangerous)
  -P, --preserve     - move package contents to preserve dir
  -r, --root <path>  - use an alternative remove path
  -v, --verbose      - be more verbose
  -V, --version      - print name, version and license
  -h, --help         - print this message and exit
[root@localhost:~]#

The detailed explanation of the options is here:

-b,--nobackup

    when a package is removed pkg_remove will check if the package that is about to be removed contains files marked as BACKUPS (usually configuration files) and save the files as FILE.pkgbackup, using this option makes pkg_remove delete these files too
-d,--nodeps
    when you remove a package pkg_remove will check if other packages depend on the package that you are about to remove, if such packages exist, then pkg_add will exit with an error message. using the option will make pkg_remove skip this step (this option cannot be used together with the -c option).
-P, --preserve
    this option will make pkg_remove save the contents of the package that is about to be removed in the $PRESERVEDIR (defined in /etc/dpack/common), usually this is /var/lib/pkg/preserve/PKGNAME. This is usefull when you want to keep the package just in case something doesnt work well with your next step.
-r, --root <path>
    use an alternate <path> for removing. This option should be used if you want to remove packages from another system (eg: a amounted partition).
-v, --verbose
    be more verbose, use this option if you want pkg_remove to print more information (without this option pkg_remove will only print a one line message)
Examples:

Removing a package:
[root@localhost:~]# pkg_remove ed
[+] removing ed...[OK]
[root@localhost:~]#

Removing a package (verbose mode):
[root@localhost:~]# pkg_remove -v ed
[+] removing ed...
   [-] checking dependencies...[ OK ]
   [-] saving backup files...[ DONE ]
   [-] removing files...
        /bin/ed
        /usr/bin/red
        /usr/share/doc/ed-0.2/ChangeLog
        /usr/share/doc/ed-0.2/COPYING
        /usr/share/doc/ed-0.2/INSTALL
        /usr/share/doc/ed-0.2/NEWS
        /usr/share/doc/ed-0.2/POSIX
        /usr/share/doc/ed-0.2/README
        /usr/share/doc/ed-0.2/THANKS
        /usr/share/doc/ed-0.2/TODO
        /usr/share/info/ed.info.gz
        /usr/share/man/man1/ed.1.gz
        /usr/share/man/man1/red.1
        /usr/share/doc/ed-0.2/
   [-] removing database entry...[ DONE ]
[root@localhost:~]#

Removing a package (and printing dependencies errors):
[root@localhost:~]# pkg_remove gtk-xfce-engine
[+] removing gtk-xfce-engine...[FAILED]
[ERROR]: packages that depend on gtk-xfce-engine:
     libxfcegui4
[root@localhost:~]#

Removing same package (with -d options):
[root@localhost:~]# pkg_remove -d gtk-xfce-engine
[+] removing gtk-xfce-engine...[OK]
[root@localhost:~]#

The package has been removed, but programs from the package(s) that depend on it will not
work anymore, so this option should only be used if you really know what you are doing.
