Updating packages

Instead of downloading packages manually from the internet, or building them with dsrc (although you can enable special options), it is possible to install or upgrade packages "on-the-fly" with dupdate. The updating mechanism is very simple, you have a update info database in $DATABASEDIR/update/, which can be updated every day and packages that wait for you on the internet.

To install/upgrade a package you use this command:
[root@localhost:~]# dupdate -h
usage: dupdate [options] <argument1> <argument2> ...
options:
  -d, --nodeps       - skip all dependencies checks (dangerous)
  -D, --deps         - resolve dependencies before updating
  -f, --fetch        - only download packages, do not install them
  -i, --info         - show information about a package
  -m, --md5sum       - skip md5sum check
  -r, --reinstall    - reinstall the package (same version and release)
  -s, --search       - search packages that contain *argument*
  -u, --upgrade       - only upgrade installed packages (used with a group)
  -v, --verbose      - be more verbose
  -V, --version      - print name, version and license
  -H, --howto        - a short howto with some examples
  -h, --help         - print this message and exit
arguments:
  <package>    - will update the package
  <group>      - will update entire group
  <world>      - will update the entire system
  <sync>       - will update the package info files

For a detailed help please see dupdate (8).
[root@localhost:~]#

The detailed explanation for the options:

-d,--nodeps

    this will make dupdate skip all dependencies checks. This is not recommended since a package cant work if it has missing dependencies. Only use if you really know what you are doing.
-D,--deps
    normally dupdate will check if all needed dependencies are installed and if not, it will exit with an error message (the missing dependencies). If you want dupdate to install missing dependencies before installing/upgrading the package use this option.
-i, --info
    print information about a package that is available for installation with dupdate. It will show the following variables: LATEST, INSTALLED, HOMEPAGE, DESCRIPTION, LICENSE, DOWNLOAD, SIZE, DEPENDENCIES and MD5SUM.
-m, --md5sum
    normally dupdate will check package integrity (against the md5sum from the update info file) and exit its corrupted. use this option to skip this step.
-r, --reinstall
    if the package that is about to be upgraded is the same as the installed one (same version and release) dupdate will exit with a error message (package name-version-arch-release.tbz2 already installed (use --reinstall)). Use this option if you want to reinstall the package.
-s, --search
    search for a string in the package update database and print packages that contain the string.
-u, --upgrade
    only upgrade packages that are out of date (this should be used when you update a group.
-v, --verbose
    be more verbose (self explanatory).
